Power Realms — Полная концепция (расширенная версия)
1. Цель и миссия

Power Realms — это децентрализованная платформа для объединения вычислительных ресурсов пользователей, позволяющая:

Пользователям с мощными ПК сдавать свои мощности в аренду.

Пользователям с менее мощными ПК арендовать мощности для игр и приложений.

Организовывать пулы с автономной экономикой, где участники могут зарабатывать, распределять доходы и выводить средства внутри пула (Off-platform) без участия платформы.

Предоставлять GameBoost для игр с высокими требованиями, распределяя мощность пула.

Поддерживать майнинг, рендеринг, вычислительные задачи и другие нагрузки через совместные ресурсы пула.

2. Пользователи и роли
Роль	Возможности	Ограничения
Global Admin	Первый зарегистрированный пользователь, настройки платформенных комиссий, реклама, мониторинг пулов	Не управляет внутренними выплатами
Pool Owner	Создание и управление пулом, Marketplace пула, подтверждение Off-platform выплат, распределение доходов	Должен соблюдать правила платформы и прозрачность ledger
Officer	Поддержка Owner, арбитраж споров, подтверждение Hold и Off-platform выплат	Не может менять платформенные комиссии
Member	Предоставление ресурсов, аренда GameBoost слотов, участие в майнинге, marketplace, запрос на вывод средств	Не может изменять правила пула без согласия Owner
3. Пулы
3.1 Типы пулов

Публичные: любой пользователь может присоединиться.

Приватные: доступ только по паролю.

Премиум: платные для создания, расширенные возможности, повышенные лимиты на машины и слоты, доступ к аналитике.

3.2 Управление пулом

Owner управляет правилами, лимитами, Marketplace и распределением доходов.

Officer решает споры и помогает контролировать операции.

Members предоставляют ресурсы и получают доход.

3.3 Ограничения участников

Default: максимум 10 машин на участника.

+1 машина за 5 приглашённых друзей (реферальная система) или покупка через premium.

Возможность ограничивать покупки и аренду внутри пула для балансировки экономики.

4. Внутренняя экономика
4.1 Кошельки и Ledger

MemberBalance: внутренний баланс участника, используется для всех операций внутри пула (аренда мощности, GameBoost, косметика, майнинг).

PoolLedger: фиксирует все транзакции, Hold/Escrow, распределение доходов и Off-platform withdrawals.

CustodialPlatformWallet: принимает fiat/USDT для пополнения MemberBalance; не используется для вывода средств.

4.2 Потоки средств

Пополнение MemberBalance: fiat → PlatformWallet → PoolLedger → MemberBalance

Покупка Compute Time/GameBoost: MemberBalance → Hold → MemberBalance продавца после выполнения

Reward distribution: Owner → PoolLedger → MemberBalance

Off-platform Withdrawals: MemberBalance → Hold → Owner/Officer подтверждает → Ledger фиксирует Completed

5. Marketplace
5.1 Товары и услуги

Compute Time: CPU/GPU часы для участников.

GameBoost Slots: аренда мощности для игр.

Private Slot Access: доступ к закрытым или премиум пулам.

Cosmetics / Tier Upgrades: визуальные улучшения и привилегии.

5.2 Механика

Участники публикуют предложения с ценой в MemberBalance.

Hold/Escrow блокирует средства до завершения сделки.

Owner регулирует лимиты и минимальные цены.

Ledger фиксирует все операции.

6. GameBoost
6.1 Принцип работы

Seeker (слабый ПК) подключается к Node (мощная машина) через WebRTC.

Node запускает игру, Seeker получает видеопоток.

Hold блокирует оплату до завершения сессии.

После завершения: Node получает оплату, metrics фиксируются.

6.2 SLA и рейтинг

Метрики: latency, FPS, uptime.

Partial refund при плохом качестве.

Рейтинг Node влияет на доступность и цены слотов.

7. Майнинг внутри пула

Пулы поддерживают совместный майнинг: CPU/GPU участников объединяются для решения задач.

MemberBalance начисляется пропорционально предоставленной мощности.

Reward distribution: Owner распределяет вознаграждение между участниками.

Hold/Escrow используется для временной блокировки доходов до подтверждения выполнения работы.

Можно включить специализированные задачи: рендеринг, AI computation, научные вычисления.

8. Hold / Escrow

Hold блокирует средства до завершения сделки или выполнения задачи.

После выполнения: Hold → MemberBalance продавца/Node с удержанием комиссии.

Dispute Flow используется для возврата средств при проблемах.

Ledger фиксирует статус: Pending → Released → Completed.

9. Off-platform Withdrawals

Member инициирует Withdrawal Request.

Hold блокирует сумму.

Owner/Officer подтверждает перевод на внешний кошелёк.

Ledger фиксирует транзакцию: Requested → Confirmed → Completed.

Возможности: ручной перевод Owner, multisig/escrow автоматизация, Buyback между участниками пула.

10. Dispute Flow

Инициатор: Member или Node.

Arbiters: Owner и Officers.

Платформа фиксирует доказательства и решение.

Возможные исходы: partial refund, full refund, payout внутри пула.

Ledger фиксирует спор: Open → Resolved → Completed.

11. Рейтинг и репутация

Nodes: uptime, quality metrics, SLA.

Members: вклад в пул, reliability.

Плохой рейтинг → ограничения доступа, приостановка выплат или аренды.

Anti-fraud: wash trading detection, лимиты, мониторинг IP/HWID.

12. UX / интерфейс

Wallet UI: баланс, pending holds, история, Off-platform Withdrawals.

Marketplace UI: листинг предложений, Buy/Reserve, Hold, Completion.

GameBoost UI: выбор ноды, аренда слота, подключение, завершение.

Notifications: запросы на вывод, завершение сессии, арбитраж.

13. Техническая реализация

Endpoints: create hold, accept offer, start GameBoost session, request withdrawal, dispute, settle rewards.

Events: HoldCreated, HoldReleased, SessionStarted, SessionEnded, WithdrawalRequested, WithdrawalConfirmed, DisputeOpened, DisputeResolved.

Ledger фиксирует все операции, включая Off-platform Withdrawals.

WebRTC для GameBoost, REST/GraphQL API для пула и marketplace.

Multisig/escrow wallet поддержка для автоматизации выплат.

14. Монетизация

Комиссия с внутренних транзакций (10%).

Ads revenue: показы и рекламные кампании.

Premium features: шаблоны пулов, аналитика, boosted discovery.

Продажа дополнительных слотов: доход владельца + комиссия платформы.

Платные премиум-пулы: Owner получает дополнительный доход.

15. Финансовая модель

Внутренние транзакции: комиссия 10%.

Ads revenue зависит от активности пользователей.

Premium features и платные пулы — дополнительный доход.

Off-platform Withdrawals — платформа фиксирует только транзакции.

16. Roadmap

Phase 0: PoC PoolLedger, Marketplace, GameBoost, майнинг (3 мес)

Phase 1: MVP public beta (6 мес)

Phase 2: Ads, Admin tools, Dispute workflow (9–12 мес)

Phase 3: Enterprise features, cross-pool trades, Off-platform withdrawal automation (12–24 мес)

17. Примеры сценариев

Member присоединяется к публичному пулу, предоставляет мощности для майнинга → зарабатывает MemberBalance → вывод средств через Owner.

Премиум-пул собирает взносы, распределяет доход внутри пула → участники используют GameBoost или продают Compute Time → Off-platform вывод средств.

Seeker арендует GameBoost слот → Node получает оплату → рейтинг Node обновляется.

Dispute по низкому качеству GameBoost → Officer разрешает, partial refund.

18. Безопасность

Hold/Escrow для защиты сделок.

Multisig для пулевых выплат.

Anti-fraud: wash trading detection, лимиты на машины и покупки.

Ledger фиксирует все операции, прозрачность для участников.

19. Заключение

Полностью автономные экономики внутри пулов.

Off-platform Withdrawals без участия платформы.

GameBoost, майнинг, Marketplace, Hold/Escrow, Dispute flow интегрированы.

Платформа фиксирует транзакции, обеспечивает прозрачность и безопасность.

Масштабируемая и юридически безопасная система, гибкая для премиум и приватных пулов.
